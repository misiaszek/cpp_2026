1. std::array – Bezpieczniejsza alternatywa dla tablic C

W tradycyjnym C tablice są problematyczne: "gubią" swój rozmiar przy przekazywaniu do funkcji (degradacja do wskaźnika) i nie oferują metod pomocniczych.

Dlaczego warto używać std::array?

  • Zna swój rozmiar - Metoda size() zwraca liczbę elementów.
  • Bezpieczeństwo - Metoda at() sprawdza zakres (wyrzuca wyjątek przy błędzie).
  • Wydajność - Pamięć jest alokowana na stosie (tak jak w tablicy C), brak narzutu dynamicznego.
  • Interfejs STL - Współpracuje z iteratorami i algorytmami standardowymi.
#include <array> std::array<int, 5> n = { 32, 27, 64, 18, 95 }; // Typ + Rozmiar

2. Rozmiar musi być stały (Compile-time)

Rozmiar std::array jest częścią jego typu. Musi być znany już w momencie kompilacji programu.

Używaj const size_t lub (lepiej w nowoczesnym C++) constexpr size_t do definiowania rozmiarów tablic.
const size_t arraySize = 5; std::array s; // Poprawnie // C++17: Class Template Argument Deduction (CTAD) std::array data = { 1, 2, 3 }; // Kompilator sam wywnioskuje !

Wypełnianie tablicy w pętli pozwala na dynamiczne obliczenie wartości początkowych, mimo stałego rozmiaru kontenera.

3. Formatowanie wyjścia i Wizualizacja danych

Biblioteka <iomanip> pozwala na tworzenie czytelnych tabel i raportów w konsoli.

  • std::setw(n) - Ustala szerokość pola dla następnego elementu.
  • std::left / std::right - Wyrównanie tekstu.

Przykład: Tekstowy wykres słupkowy

for (size_t i = 0; i < n.size(); ++i) { std::cout << std::setw(7) << i << ": "; for (unsigned int stars = 0; stars < n[i]; ++stars) std::cout << '*'; std::cout << std::endl; }

Indeksy tablicy mogą reprezentować kategorie (np. przedziały ocen), a wartości – liczebność tych kategorii.

4. Nowoczesne Losowanie Liczb (<random>)

Zapomnij o rand() % n. Jest to metoda przestarzała, o słabym rozkładzie i małym zakresie.

C++11 wprowadził bibliotekę <random>, która oddziela silnik generujący bity od rozkładu prawdopodobieństwa.
#include <random> std::default_random_engine engine(time(0)); // Silnik (ziarno) std::uniform_int_distribution randomInt(1, 6); // Rozkład (kostka) int wynik = randomInt(engine); // Generowanie wartości

Zalety: Lepsza jakość losowości, łatwa zmiana rozkładu (np. na normalny/Gaussa), brak "skrzywienia" modulo.

5. Czas życia obiektu: static vs automatic

Zmienne lokalne (w tym tablice std::array) mogą mieć różny czas życia (storage duration), co drastycznie zmienia ich zachowanie.

Automatyczne (Domyślne)

  • Tworzone na stosie przy każdym wejściu do funkcji.
  • Niszczone przy wyjściu (klamra }).
  • Brak inicjalizacji: Jeśli nie podasz wartości, w tablicy mogą być "śmieci" (nieokreślone dane).
  • Kosztowne przy dużych rozmiarach (re-inicjalizacja).

Statyczne (static)

  • Tworzone w segmencie danych raz na cały czas działania programu.
  • Inicjalizacja zerami: Jeśli nie podasz wartości, static std::array zostanie automatycznie wyzerowana przy starcie.
  • Pamięć: Zachowuje wartości między wywołaniami funkcji.
  • Wydajne dla dużych, niezmiennych tablic (lookup tables).
void initDemo() { static std::array stat; // Wszystko = 0, inicjalizacja raz! std::array auto_arr; // Wartości nieokreślone (garbage)! stat[0] += 5; // Przy kolejnym wywołaniu będzie tu 5, 10, 15... }

6. Range-based for (Pętla zakresowa)

Wprowadzona w C++11, pozwala na bezpieczną i zwięzłą iterację po wszystkich elementach kontenera bez używania indeksów.

std::array items = { 1, 2, 3, 4, 5 }; // TYLKO ODCZYT (bezpiecznie przez referencję do const) for (const auto& item : items) { std::cout << item << " "; } // MODYFIKACJA (przez referencję) for (auto& item : items) { item *= 2; }
Używaj auto& dla wydajności (brak kopiowania) i bezpieczeństwa.

7. C++17: Udogodnienia dla tablic

  • std::size() - Globalna funkcja zwracająca rozmiar kontenera (lub tradycyjnej tablicy!). Bardziej uniwersalna niż .size().
  • CTAD - Wspomniane wcześniej wnioskowanie typów dla szablonów klas (std::array a = {1, 2};).
  • Zmienne strukturalne (Structured Bindings) - Pozwalają "rozpakować" elementy tablicy do nazwanych zmiennych (choć częściej używane z pair lub struct).
// Przykład std::size() int oldTable[] = {1, 2, 3}; auto s = std::size(oldTable); // Działa dla tablic C i std::array!
