1. Destruktor: Sprzątanie po obiekcie

W Laboratorium 3 poznaliśmy konstruktory.

  • Nazwa: ~NazwaKlasy() (z tyldą).
  • Wywoływany automatycznie, gdy obiekt ginie.
  • Brak argumentów i typu zwracanego.
  • Służy do zwalniania zasobów (RAII).

2. Przykład Destruktora (Kod)

class Simple { int m_id; public: Simple(int id) : m_id{id} { cout << "Tworzę " << m_id << "\n"; } ~Simple() { cout << "Niszczę " << m_id << "\n"; } };

3. Czas życia na stosie (Stack)

Obiekty lokalne żyją tylko wewnątrz bloku { }.

{ Simple s1{1}; { Simple s2{2}; } // s2 ginie TUTAJ } // s1 ginie TUTAJ

Obiekty są niszczone w odwrotnej kolejności.

4. Rodzaje obiektów a cykl życia

W cw7-2 widzimy różne typy obiektów:

  • Globalne: Tworzone przed main().
  • Statyczne: Żyją do końca programu.
  • Automatyczne: Żyją do końca bloku.
  • Dynamiczne: Żyją do momentu delete.

6. Obiekty stałe (const)

Słowo const chroni obiekt przed zmianą.

  • Może wywoływać tylko metody const.
  • Metoda obiecuje, że nie zmieni stanu.
const Time noon(12, 0, 0); noon.printUniversal(); // OK noon.setHour(13); // BŁĄD KOMPILACJI

7. Funkcje składowe const

Metodę oznaczamy po nawiasach parametrów.

class Time { int hour; public: // ODCZYT (bezpieczna) int getHour() const { return hour; } // ZMIANA (brak const) void setHour(int h) { hour = h; } };

8. Pułapka braku const

Brak const blokuje wywołanie dla obiektów stałych.

void print() { cout << hour; } // Brak const! const Time t(10); t.print(); // BŁĄD kompilacji!
💡 Zawsze dodawaj const do metod odczytujących.

9. Wskaźnik "this"

W każdej metodzie istnieje ukryty wskaźnik this.

  • Wskazuje na bieżący obiekt.
  • Rozróżnia pola klasy od parametrów.
void setX(int x) { this->x = x; }
Wydział Fizyki, Astronomii i Informatyki Stosowanej UJ

10. Trzy sposoby na to samo

Zapisy równoważne wewnątrz metody:

  1. x (niejawne)
  2. this->x (jawne)
  3. (*this).x (dereferencja)

Odnoszą się do pola bieżącego obiektu.
