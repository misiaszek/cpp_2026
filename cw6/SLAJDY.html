1. Bezpieczny wektor - metoda at()

W Laboratorium 5 poznaliśmy std::vector.

  • v[i] - Szybki, ale brak kontroli zakresu.
  • v.at(i) - Bezpieczny (sprawdza zakres).
  • Błędny indeks -> rzuca std::out_of_range.
std::vector data = {1, 2, 3}; try { data.at(10) = 666; // Błąd! } catch (const std::out_of_range& e) { std::cout << e.what(); }

2. Mechanizm Wyjątków

Wyjątki reagują na awarie w kontrolowany sposób.

  • throw - zgłoszenie błędu.
  • try - blok "obserwowany".
  • catch - obsługa konkretnego typu.
try { throw 42; } catch (int e) { std::cout << "Złapano: " << e; }

3. Hierarchia Wyjątków

Większość błędów dziedziczy po std::exception.

  • std::out_of_range - błąd indeksu.
  • std::runtime_error - błąd wykonania.
  • catch(...) - łapie wszystko.
try { ... } catch (const std::exception& e) { std::cerr << e.what(); } catch (...) { std::cerr << "Nieznany błąd!"; }

6. Organizacja kodu: Pliki .h

Dzielimy projekt na mniejsze części.

  • .h (header) - deklaracje klas i funkcji.
  • .cpp - implementacja metod.
  • Dołączamy przez #include "Plik.h".

8. Nowoczesna Alternatywa: #pragma once
