SLAJD 1: Struktury w stylu C vs C++

W C++ struktura to coś więcej niż kontener na dane – może posiadać zachowania (metody).

// Styl C++ (cw2-2.cpp) struct Date { int year, month, day; void print() { // Metoda wewnątrz struktury std::cout << year << "/" << month << "/" << day; } }; Date today { 2020, 10, 14 }; today.print(); // Składnia obiektowa
  • Wniosek: Dane i logika są teraz ze sobą powiązane.
Pliki: cw2-1.cpp, cw2-2.cpp

SLAJD 2: Narodziny Klasy (class)

Klasa wprowadza domyślną prywatność, co jest fundamentem enkapsulacji.

class Date { public: // Interfejs dostępny dla wszystkich void print() const; private: // Dane ukryte przed światem zewnętrznym int m_year { 2020 }; int m_month { 10 }; int m_day { 14 }; };
  • Struct: domyślnie publiczny (wszystko widać).
  • Class: domyślnie prywatna (wszystko ukryte).
  • Konwencja m_ ułatwia identyfikację pól klasy.
Plik: cw2-3.cpp

SLAJD 3: Gettery i Settery

Kontrolowany dostęp do prywatnych danych przez publiczne metody.

// Getter: tylko odczyt (const) int getYear() const { return m_year; } // Setter: zmiana wartości (miejsce na walidację) void setYear(int y) { m_year = y; }
  • Chroni dane przed przypadkowym uszkodzeniem.
  • Pozwala na zmianę wewnętrznej implementacji bez wpływu na resztę programu.
Plik: cw2-4.cpp

SLAJD 4: Spójność Danych (Inwarianty)

Klasa pilnuje, aby powiązane ze sobą dane były zawsze poprawne.

void setName(std::string_view name) { m_name = name; m_firstInitial = name.front(); // Automatyczna spójność! }
  • Uniemożliwia powstanie obiektów niespójnych (np. Imię "Jan", a inicjał "K").
  • std::string_view (C++17) zapewnia szybki dostęp do tekstu bez kopiowania.
Plik: cw2-5.cpp

SLAJD 5: Konstruktory

Specjalna metoda wywoływana automatycznie podczas tworzenia obiektu.

// Pułapka (cw2-6.cpp): Foo(int x, int y) { // x i y to tylko parametry, nie zmieniają pól m_x i m_y! } // Rozwiązanie (cw2-7.cpp): Foo(int x, int y) : m_x{x}, m_y{y} // Lista inicjalizacyjna { }
  • Konstruktor nie ma typu zwracanego.
  • Nazywa się tak samo jak klasa.
Pliki: cw2-6.cpp, cw2-7.cpp

SLAJD 6: Lista Inicjalizacyjna

Najbardziej efektywny i poprawny sposób inicjalizacji pól w C++.

Foo(int x, int y) : m_x{x}, m_y{y} { // Ciało konstruktora zazwyczaj pozostaje puste }
  • Inicjalizuje pola bezpośrednio przed wywołaniem konstruktora.
  • Wymagana dla pól stałych (const) i referencji.
  • Zapobiega podwójnej pracy kompilatora (inicjalizacja domyślna + przypisanie).
Standard: Modern C++

SLAJD 7: Podsumowanie

  • Enkapsulacja: Dane prywatne, metody publiczne.
  • Bezpieczeństwo: Gettery oznaczaj jako const.
  • Inicjalizacja: Używaj list inicjalizacyjnych w konstruktorach.
  • Czytelność: Stosuj konwencję m_nazwaPola.
